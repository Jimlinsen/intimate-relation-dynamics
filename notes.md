# Notes: 沦陷度检测指导系统 — 设计研究

## 现有资产盘点

### 三套理论框架

| 框架 | 核心模型 | 核心变量 | 适用场景 |
|------|---------|---------|---------|
| 猜疑链动力学 | E = S × I × Δ | S(猜疑张力) I(不可逆浓度) Δ(变化率) | 快速判断、实战指导 |
| τ自指循环周期 | F_R = D_want × D_feedback × (1+min(I)/I₀) | τ_her τ_you D_want D_feedback I_n Δτ | 精确测量、节奏诊断 |
| 关系力学全景 | 六维+四象限+纵向因果 | Δτ ℛ(Q1-Q4) I_n Σ_debt θ_OC D_want + C_V/C_H | 深度诊断、结构性分析 |

### 三个现有组件

| 组件 | 文件 | 特点 | 输出格式 |
|------|------|------|---------|
| 沦陷度实测v2 | 沦陷度实测v2.jsx | 3维度,信号强度,沦陷等级 | S/I/D bars + signal + label |
| τ关系测度 | tau关系测度.jsx | 5变量,τ诊断,同步建议 | 5 bars + phase + Δτ card |
| 关系力测度 | 关系力测度.jsx | 6维+4象限+因果分层 | quad chart + causal + sigma |

### 共性模式（可复用）

1. **交互流程一致**: input → mission → report → result → next round
2. **API调用模式一致**: callAI(messages) → parseJSON(text) → render
3. **视觉语言一致**: 深色背景 #0b0a09, 主色 #e8927c, 卡片式布局
4. **AI 交互模式**: 给任务JSON → 用户执行 → 汇报反应 → 评估JSON

### 理论层级关系（关键洞察）

```
         关系力学全景（完整诊断框架）
              ↗           ↖
    τ理论（时间动力学）   四象限（空间结构）
         ↑                    ↑
    猜疑链动力学          纵向因果分析
    （直觉入口）         （深层机制）
```

三套理论不是并列的，是**层级递进**的：
- 猜疑链动力学 = 入口层（直觉可用，门槛低）
- τ理论 = 中间层（增加时间维度的精确测量）
- 关系力学 = 完整层（加上空间结构 + 自指消化 + 纵向因果）

## 用户画像推断

- 正在经营或想发展一段亲密关系
- 需要可执行的行动指导（话术、行动、观察点）
- 能接受理论框架但不需要时时看到推导过程
- 使用场景：手机上操作，边聊天边用这个工具

## 关键设计洞察

1. **统一不是拼接**: 不是把3个app放到tab里，而是用1个统一的诊断流程，背后根据情况调用不同层级的理论
2. **渐进揭示**: 用户看到的是简洁的结果，点开才看到深层诊断
3. **累积智能**: 每轮探测的数据应该积累，形成越来越精确的关系画像
4. **双向适配**: 系统根据已有数据自动选择下一个最有价值的探测维度

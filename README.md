# 沦陷度检测指导系统

亲密关系的实时诊断仪 + 行动导航仪。

基于三套原创理论框架，通过 AI 驱动的交互式探测，量化亲密关系的动态结构，并给出可直接执行的行动指导。

## 理论基础

### 框架一：猜疑链动力学

> 爱的源头不是匮乏，是猜疑链。

**核心公式：E = S × I × Δ**

| 变量 | 含义 | 健康区间 |
|------|------|---------|
| **S** (Suspicion) | 猜疑张力 — 对方模型中未被解释的你的比例 | 0.3 ~ 0.7 |
| **I** (Irreversibility) | 不可逆浓度 — 不可撤回行动占全部互动的比例 | 0.15 ~ 0.2 |
| **Δ** (Delta) | 变化率 — 你自身的重建频率与打补丁频率之比 | > 1 |

三个变量中任何一个归零，E 归零，爱情熄火。S 和 I 形成跷跷板：`I_target = 0.2 × (1 + (0.5 - S))`。

**三种死亡螺旋：**
- **无聊之死**：Δ 降低 → S 趋零 → 无张力 → E 归零
- **焦虑之死**：S 过高 → I 不足以锚定 → 防御性建模 → 猜疑变破坏力
- **窒息之死**：I 过高 → 沉没成本累积 → Δ 被迫降低 → 进入无聊之死

### 框架二：τ 自指性循环周期

> τ(x, D) = 对象 x 在环境 D 中从被提出到被反弹所需的时间。

**权力方程：** `P(A, B) = τ(B→A) / τ(A→B)`

P > 1 则 A 对 B 有权力。权力不由能力决定，由 τ 差决定。

**关系动力学方程：** `dF_R/dt = λ(D_want · D_feedback - F_R) - γ · Δτ²`

- Δτ 不对称始终消耗关系力，使关系趋于衰减
- 修复关系不是"多说话"，是"同步回应节奏"（校准 τ）
- 恋爱阶段的 τ 特征：初期双短对称 → 稳定期延长对称 → 危机期出现不对称

**意向梯度 I_n：**

```
I_1: A 需要 B
I_2: B 知道 A 需要 B → B 调整回应
I_3: A 知道 B 知道 → A 调整表达（博弈开始）
I_4: B 知道 A 知道 B 知道…（博弈加深）
```

`min(I_A, I_B)` 决定关系的最大复杂度。

### 框架三：关系力学全景

**四维关系空间 ℛ：**

|  | 直接 | 间接 |
|---|---|---|
| **现实** | Q_Ⅰ 身体场（物理在场、接触） | Q_Ⅱ 制度场（共同习惯、物质绑定） |
| **虚拟** | Q_Ⅲ 直觉场（化学反应、默契） | Q_Ⅳ 价值场（世界观对齐、精神共鸣） |

**自指守恒律：** `Σ_stage + Σ_debt = Σ_total`

不做阶段性消化（健康冲突），自指债务就会积累，直到全面崩溃。表面稳定不等于结构性健康 — 一段从不吵架的关系可能 Σ_debt 极高。

**纵向因果 C_V：** 高层级的什么结构，蕴含了低层级现象的必然出现？

> "他不回消息"的横向原因可能是忙。纵向原因可能是：τ 结构已失衡 → θ_OC 严重偏向秩序 → Σ_debt 长期积累。修复关系不是在横向因果层面"多发消息"，而是在纵向因果层面校准结构。

## 系统功能

三种诊断模式，层级递进：

| 模式 | 理论层 | 探测维度 | 适用场景 |
|------|-------|---------|---------|
| **快速** | E = S × I × Δ | 猜疑张力、不可逆浓度、变化感知 | 直觉判断，快速出结果 |
| **标准** | τ 测度 | τ她/τ你、欲求值、回馈质量、博弈深度 | 精确测量回应节奏 |
| **深度** | 关系力全景 | 6维 + 4象限 + 纵向因果 | 结构性诊断 |

**交互流程：**

```
描述关系背景 → 选择诊断模式 → 接收探测任务（话术+行动+观察点）
     ↑              → 执行并汇报对方反应 → 获得诊断结果与行动建议      │
     └──────────────── 多轮迭代，画像越来越精确 ─────────────────────┘
```

**统一仪表盘：** 跨模式数据聚合，F_R 趋势追踪，核心指标一目了然。

## 快速开始

```bash
# 克隆
git clone https://github.com/Jimlinsen/intimate-relation-dynamics.git
cd intimate-relation-dynamics

# 配置 API Key
cp .env.example .env
# 编辑 .env，填入你的 API Key

# 安装 & 启动
npm install
npm run dev
```

打开 http://localhost:3210

## 技术栈

- **前端：** React 19 + Vite
- **AI：** 大语言模型 API（OpenAI 兼容格式）
- **架构：** 纯提示工程，理论框架编码进 System Prompt，无需 RAG/微调

## 一条边界

> 所有模型都不对，但有些模型有用。

这个系统的有效边界：仅适用于双方都有基本好感的前提下。S、I、Δ 的具体数值无法精确测量，只能做定性判断。量化是为了建立直觉，不是为了计算。

模型是骨架，真实是血肉。填充模型的内容，只能是你这个人本身。
